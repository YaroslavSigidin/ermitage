# Настройка авто-деплоя на Timeweb (один раз)

Чтобы при каждом `git push` сайт автоматически заливался на Timeweb, нужно один раз добавить данные FTP в секреты GitHub.

## Шаг 1. Открой настройки репозитория

1. Зайди на https://github.com/YaroslavSigidin/ermitage  
2. Нажми вкладку **Settings** (Настройки)  
3. В левом меню выбери **Secrets and variables → Actions**

## Шаг 2. Добавь три секрета

Нажми **New repository secret** и создай по одному секрету для каждого значения:

| Name (имя)   | Value (значение)   | Описание        |
|--------------|--------------------|-----------------|
| `FTP_SERVER` | `62.113.41.14`     | IP твоего сервера |
| `FTP_USERNAME` | `wj677987`       | Логин FTP       |
| `FTP_PASSWORD` | `wnL_bEXJ1TS,W7` | Пароль FTP      |

- Имя вводи **точно** как в таблице (с большими буквами и подчёркиванием).  
- Значение вставляй без лишних пробелов.

## Шаг 3. Сохрани и запушь workflow

В проекте уже есть папка `.github/workflows/` с файлом `deploy-timeweb.yml`. Нужно отправить его на GitHub:

```bash
cd "/Users/sigidingo/Desktop/КУРСОР/restaurant-menu"
git add .github/
git commit -m "Автодеплой на Timeweb при пуше"
git push
```

После первого пуша с этим файлом зайди на вкладку **Actions** в репозитории — там должен запуститься workflow «Deploy to Timeweb». Если секреты добавлены верно, сайт зальётся на сервер.

## Дальше

При каждом **push** в ветку **main** GitHub сам загрузит файлы на Timeweb в папку `public_html`.  

Проверять деплой: **Actions** → выбери последний запуск «Deploy to Timeweb» → смотри логи и статус (зелёная галочка = успех).
