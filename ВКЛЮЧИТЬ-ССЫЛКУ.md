# Одна рабочая ссылка на меню (GitHub Pages)

Деплой на Timeweb идёт, но по IP страница не открывается — так устроен хостинг.  
Проще всего получить **одну рабочую ссылку** через GitHub Pages (бесплатно, без настроек сервера).

---

## Твоя ссылка будет такой

**https://yaroslavsigidin.github.io/ermitage/**

(обязательно со слэшем в конце)

---

## Что сделать один раз (2 минуты)

1. Открой в браузере:  
   **https://github.com/YaroslavSigidin/ermitage/settings/pages**

2. В блоке **«Build and deployment»**:
   - у **Source** выбери **«Deploy from a branch»**;
   - у **Branch** выбери **main** и папку **/ (root)**;
   - нажми **Save**.

3. Подожди **1–2 минуты**.

4. Открой в браузере:  
   **https://yaroslavsigidin.github.io/ermitage/**

Страница меню должна открыться. При каждом `git push` в **main** она будет обновляться сама.

---

## Если ссылка не открывается

- Подожди ещё 2–3 минуты и обнови страницу (Ctrl+F5 или Cmd+Shift+R).
- Убедись, что открываешь именно **https://yaroslavsigidin.github.io/ermitage/** (с `https`, без опечаток).
- Зайди снова на **https://github.com/YaroslavSigidin/ermitage/settings/pages** — сверху должно быть зелёное сообщение с этой ссылкой, когда сайт уже опубликован.

---

## Домен потом

В разделе **Pages** (тот же Settings → Pages) можно добавить свой домен — GitHub покажет, какие DNS-записи создать. Тогда меню будет открываться и по твоему домену.
