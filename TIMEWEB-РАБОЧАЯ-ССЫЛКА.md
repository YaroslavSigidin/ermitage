# Как получить рабочую ссылку на Timeweb и привязать домен

Файлы на сервер заливаются (деплой зелёный), но сайт по IP не открывается — обычно это из‑за настроек веб‑сервера в панели. Ниже что сделать по шагам.

---

## Шаг 1. Узнать, куда попали файлы

После каждого деплоя в GitHub открой **Actions** → последний запуск **Deploy to Timeweb** → шаг **«Проверка содержимого на сервере»**. В логе будет:
- `pwd` — папка, в которую ты «попадаешь» по SFTP (корень загрузки);
- `ls -la` — что лежит в этой папке (должны быть `index.html`, `assets` и т.д.).

Запомни или скопируй этот путь (например `/home/wj677987` или похожий).

---

## Шаг 2. В панели Timeweb — где настраивается сайт

Зайди в личный кабинет Timeweb и найди раздел для управления сайтами. В зависимости от типа услуги это может быть:

- **Виртуальный хостинг** — раздел **«Сайты»** / **«Мои сайты»** / **«Файловый менеджер»**.
- **VPS/сервер** — раздел **«Сайты»** в панели (ISPmanager, Fastpanel, Vesta и т.п.) или **«Веб‑сервер»**.

Там должен быть **список сайтов** и у каждого — **корневая папка (Document root)**.

---

## Шаг 3. Сделать так, чтобы веб‑сервер отдавал файлы по IP

Нужно, чтобы для запросов по IP `62.113.41.14` веб‑сервер отдавал файлы из той папки, куда мы заливаем (та, что была в `pwd` в шаге 1).

### Вариант A: У тебя уже есть «сайт» по этому серверу

- Зайди в настройки этого сайта.
- Найди поле **«Корневая папка»** / **«Document root»** / **«Папка сайта»**.
- Поставь **ту самую папку**, которую показал `pwd` в логе (например `/home/wj677987` или путь к `public_html`/`www`, если файлы лежат там).
- Сохрани настройки.

После этого открой в браузере **http://62.113.41.14** — должна открыться наша страница.

### Вариант B: Сайта по IP ещё нет (часто бывает на VPS)

- В панели нажми **«Добавить сайт»** / **«Создать сайт»**.
- В качестве **домена или адреса** укажи **IP**: `62.113.41.14` (если панель такое разрешает) или временное имя, например `62.113.41.14.nip.io`.
- В **корневой папке** укажи путь из шага 1 (куда реально заливаются файлы по SFTP).
- Сохрани.

Потом открой **http://62.113.41.14** (или тот адрес, который указал).

### Вариант C: В панели можно привязать только домен

Тогда:
- Добавь сайт и привяжи к нему **домен** (например `ermitage.ru` или любой, который укажешь в DNS на этот сервер).
- В **корневой папке** укажи путь из шага 1.
- В DNS у домена укажи A‑запись на `62.113.41.14`.

Рабочей ссылкой тогда будет **http://твой-домен.ru**, позже можно включить HTTPS и привязать другой домен по той же инструкции.

---

## Шаг 4. Привязать свой домен (позже)

Когда захочешь открывать сайт по своему домену:

1. В панели Timeweb: **«Домены»** → **«Добавить домен»** → введи домен (например `ermitage.ru`).
2. Привяжи домен к **тому же сайту**, у которого уже настроена корневая папка из шага 3.
3. У регистратора домена в DNS укажи:
   - тип **A**, имя **@**, значение **62.113.41.14**;
   - при необходимости то же для **www** или CNAME для **www** на основной домен.

Через 5–30 минут сайт откроется по **http://твой-домен.ru**. Позже в панели можно включить SSL (HTTPS).

---

## Если не получается

Напиши:
- какой у тебя тип услуги (виртуальный хостинг или VPS);
- как называется панель (Timeweb, ISPmanager, Fastpanel и т.д.);
- что именно показывает шаг **«Проверка содержимого на сервере»** (скопируй строки с `pwd` и `ls -la` из лога).

По этому можно будет точно сказать, какой путь поставить в «Корневая папка» и как получить рабочую ссылку (по IP или по домену).
